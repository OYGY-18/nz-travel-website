# 网页部署指南

## 方案一：GitHub Pages（推荐 - 免费且简单）

### 步骤：
1. **创建 GitHub 账号**（如果还没有）
   - 访问 https://github.com 注册账号

2. **创建新仓库**
   - 点击右上角 "+" → "New repository"
   - 仓库名：`nz-travel-website`（或任意名称）
   - 选择 Public（公开）
   - 勾选 "Add a README file"
   - 点击 "Create repository"

3. **上传文件到 GitHub**
   - 在仓库页面点击 "uploading an existing file"
   - 将以下文件拖拽上传：
     - `index.html`
     - `styles.css`
     - `script.js`
     - `图片` 文件夹（整个文件夹）
   - 点击 "Commit changes"

4. **启用 GitHub Pages**
   - 进入仓库的 Settings（设置）
   - 左侧菜单找到 "Pages"
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main" 或 "master"，文件夹选择 "/ (root)"
   - 点击 Save

5. **获取访问链接**
   - 几分钟后，在 Pages 设置页面会显示你的网站地址
   - 格式：`https://你的用户名.github.io/nz-travel-website/`
   - 任何人都可以通过这个链接访问你的网站！

---

## 方案二：Netlify（推荐 - 最简单，支持拖拽部署）

### 步骤：
1. **访问 Netlify**
   - 打开 https://www.netlify.com
   - 使用 GitHub 账号登录（或注册新账号）

2. **部署网站**
   - 登录后，将整个项目文件夹（包含所有文件）拖拽到 Netlify 的部署区域
   - 或者点击 "Add new site" → "Deploy manually"
   - 上传你的项目文件夹

3. **获取链接**
   - 部署完成后，Netlify 会自动生成一个链接
   - 格式：`https://随机名称.netlify.app`
   - 可以自定义域名

---

## 方案三：Vercel（适合开发者）

### 步骤：
1. 访问 https://vercel.com
2. 使用 GitHub 账号登录
3. 导入你的 GitHub 仓库
4. 自动部署完成

---

## 方案四：使用内网穿透（临时测试用）

如果你想临时让别人访问本地网站：

### 使用 ngrok：
1. **安装 ngrok**
   ```powershell
   # 下载 ngrok 或使用 chocolatey
   choco install ngrok
   ```

2. **启动本地服务器**
   ```powershell
   python -m http.server 8000
   ```

3. **启动 ngrok**
   ```powershell
   ngrok http 8000
   ```

4. **分享链接**
   - ngrok 会生成一个公网链接（如：`https://xxxx.ngrok.io`）
   - 这个链接可以分享给别人访问
   - **注意**：免费版链接每次重启都会变化

## 注意事项

1. **图片路径**：确保所有图片路径使用相对路径（如：`图片/xxx.jpg`）
2. **文件大小**：GitHub 单个文件限制 100MB，仓库总大小建议不超过 1GB
3. **更新网站**：修改文件后，重新上传到 GitHub/Netlify 即可更新网站
4. **自定义域名**：所有平台都支持绑定自己的域名（需要购买域名）
