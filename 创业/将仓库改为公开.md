# 🔓 将 GitHub 仓库改为公开（启用 Pages）

## ❗ 问题

你看到的消息：
> "Upgrade or make this repository public to enable Pages"

**原因**：你的仓库是 **Private（私有）**，GitHub Pages 免费版只支持 **Public（公开）** 仓库。

---

## ✅ 解决方法：将仓库改为公开

### 步骤：

1. **进入仓库设置**
   - 在仓库页面，点击顶部的 **"Settings"**（设置）

2. **找到 Visibility 设置**
   - 向下滚动，找到 **"Danger Zone"**（危险区域）部分
   - 或者直接搜索 "Change repository visibility"

3. **修改可见性**
   - 在 "Danger Zone" 中找到 **"Change repository visibility"**（更改仓库可见性）
   - 点击 **"Change visibility"** 按钮

4. **选择 Public**
   - 会弹出确认对话框
   - 选择 **"Make public"**（设为公开）
   - 输入仓库名称确认（这是安全措施）
   - 点击 **"I understand, change repository visibility"**（我理解，更改仓库可见性）

5. **确认更改**
   - 仓库现在变为公开了
   - 刷新页面

6. **返回 Pages 设置**
   - 再次进入 **Settings → Pages**
   - 现在应该能看到部署选项了！

## ⚠️ 注意事项

### 将仓库改为公开意味着：
- ✅ 任何人都可以查看你的代码
- ✅ 任何人都可以访问你的网站
- ✅ 可以免费使用 GitHub Pages
- ⚠️ 代码是公开的（但这对静态网站通常没问题）

### 如果不想公开代码：
1. **使用 GitHub Enterprise**（付费，可以私有仓库使用 Pages）
2. **使用其他平台**：
   - Netlify（支持私有仓库）
   - Vercel（支持私有仓库）
   - Cloudflare Pages（支持私有仓库）

---

## 🎯 下一步

将仓库改为公开后：

1. **返回 Settings → Pages**
2. **现在应该能看到**：
   - "Source" 或 "Build and deployment" 选项
   - 分支选择器
   - 文件夹选择器

3. **配置部署**：
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/ (root)**
   - 点击 **Save**

4. **等待部署**（1-5分钟）

5. **访问网站**：
   - `https://你的用户名.github.io/仓库名/`

---

## 💡 提示

- 将仓库改为公开是**免费的**
- 对于静态网站（HTML/CSS/JS），公开代码通常没问题
- 如果以后需要改为私有，可以随时改回来
- 但改为私有后，GitHub Pages 会停止工作（除非升级到 Enterprise）
