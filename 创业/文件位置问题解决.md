# 📁 文件位置问题解决

## ❗ 问题

你的文件在 `创业/` 文件夹里，但 GitHub Pages 默认从仓库**根目录**查找 `index.html`。

当前结构：
```
仓库根目录/
  └── 创业/
      ├── index.html
      ├── styles.css
      ├── script.js
      └── 图片/
```

需要的结构：
```
仓库根目录/
  ├── index.html
  ├── styles.css
      ├── script.js
      └── 图片/
```

---

## ✅ 解决方案

### 方案一：把文件移到根目录（推荐）⭐

这样网站 URL 会更简洁：`https://用户名.github.io/仓库名/`

#### 方法 A：使用 GitHub 网页操作

1. **进入 `创业` 文件夹**
   - 在仓库主页，点击 `创业` 文件夹

2. **移动每个文件到根目录**
   - 点击 `index.html` 文件
   - 点击右上角的 **"✏️"**（编辑）按钮
   - 在文件路径处，删除 `创业/`，只保留文件名
   - 点击 **"Commit changes"**
   - 重复此操作移动所有文件

3. **移动文件夹**
   - 对于 `图片` 文件夹，需要：
     - 点击文件夹进入
     - 逐个移动图片文件，或
     - 使用下面的 Git 方法

#### 方法 B：使用 Git 命令行（更简单）

1. **在本地项目文件夹打开 PowerShell**

2. **如果还没有初始化 Git**：
   ```powershell
   git init
   git remote add origin https://github.com/你的用户名/仓库名.git
   git pull origin main
   ```

3. **移动文件到根目录**：
   ```powershell
   # 进入创业文件夹
   cd 创业
   
   # 移动所有文件到上一级（根目录）
   git mv index.html ../
   git mv styles.css ../
   git mv script.js ../
   git mv 图片 ../
   
   # 返回根目录
   cd ..
   
   # 提交更改
   git add .
   git commit -m "移动文件到根目录"
   git push origin main
   ```

4. **删除空的创业文件夹**（可选）：
   ```powershell
   git rm -r 创业
   git commit -m "删除空的创业文件夹"
   git push origin main
   ```

---

### 方案二：配置 Pages 从 `创业/` 文件夹部署

如果不想移动文件，可以配置 Pages 从 `创业/` 文件夹部署：

1. **进入 Settings → Pages**

2. **配置部署源**：
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/创业**（选择 `创业` 文件夹）

3. **点击 Save**

4. **访问网站**：
   - URL 会变成：`https://用户名.github.io/仓库名/创业/`
   - 或者：`https://用户名.github.io/仓库名/创业/index.html`

⚠️ **注意**：这样 URL 会包含 `/创业/`，不够简洁。

---

## 🎯 推荐：使用方案一

**原因**：
- ✅ URL 更简洁：`https://用户名.github.io/仓库名/`
- ✅ 符合 GitHub Pages 标准做法
- ✅ 更容易维护

## 📋 操作步骤（推荐方法）

### 使用 Git 命令行（最简单）：

1. **在项目文件夹打开 PowerShell**

2. **执行以下命令**：
   ```powershell
   # 如果还没有连接 GitHub
   git init
   git remote add origin https://github.com/你的用户名/仓库名.git
   git pull origin main
   
   # 移动文件
   cd 创业
   git mv * ../
   git mv 图片 ../
   cd ..
   
   # 删除空文件夹
   git rm -r 创业
   
   # 提交并推送
   git add .
   git commit -m "移动文件到根目录以支持 GitHub Pages"
   git push origin main
   ```

3. **等待几分钟**

4. **访问网站**：`https://用户名.github.io/仓库名/`
